# Plan 06-03: Exercise Management in Library

## Goal

Centralize exercise creation and configuration in the Library tab, allowing users to add and manage their exercises independently of starting a workout session.

## Requirements

- Add an "Add Exercise" button in the `LibraryView`.
- Create a new component/modal for adding/editing exercise details (name, weighted vs. bodyweight, min weight, increment).
- Display existing exercise configurations in the Library.
- Allow editing an exercise's configuration.

## Success Criteria

1.  Users can add a new exercise directly from the Library tab.
2.  Exercise configuration (min weight, increment) is preserved for each exercise.
3.  Users can edit the name and configuration of an existing exercise in the Library.
4.  Adding/editing an exercise updates the SQLite database correctly.

## Proposed Changes

### 1. `src/components/library_view.rs`

- Add an "Add Exercise" floating action button (FAB) or card.
- Implement a modal or conditional view for the "Exercise Details" form (reusing logic from `StartSessionView`).
- List exercises with their metadata and configuration (e.g., "Bench Press (45kg, 2.5kg increment)").

### 2. `src/state/db.rs` & `src/state/workout_state.rs`

- Ensure exercise updates (editing name/configuration) are supported in the database and state synchronization.

## Verification Plan

1.  Open the Library tab.
2.  Add a new weighted exercise with a specific min weight and increment.
3.  Verify the new exercise appears in the list.
4.  Edit the exercise's name and configuration, then verify the changes persist.
5.  Search for the new exercise to ensure filtering still works.
