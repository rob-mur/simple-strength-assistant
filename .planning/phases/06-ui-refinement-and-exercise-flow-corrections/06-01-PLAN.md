# Plan 06-01: BDD Test Scaffolding for UI Refinement and Flow

## Goal

Establish BDD feature files and step definitions to verify navigation positioning, exercise management in the Library tab, and the new streamlined workout start flow.

## Requirements

- BDD features for bottom navigation positioning and styling (mobile-first).
- BDD features for exercise creation and configuration in the Library tab.
- BDD features for starting a workout session from the Library's exercise detail view.
- BDD features for the empty "No active session" state in the Workout tab.

## Success Criteria

1.  All feature files created in `tests/features/`.
2.  Step definitions created or updated in `tests/steps/`.
3.  Tests compile and fail as expected (since implementation hasn't started).

## Proposed Changes

### 1. New Feature Files

- `tests/features/navigation_ui.feature`: Verifies navigation bar positioning and safe area handling.
- `tests/features/library_management.feature`: Verifies adding and editing exercises in the Library.
- `tests/features/workout_flow_new.feature`: Verifies starting a workout from the Library.

### 2. Step Definitions

- `tests/steps/navigation_steps.rs`: New step definitions for navigation checks.
- `tests/steps/library_steps.rs`: Update/create step definitions for exercise management.
- `tests/steps/workout_steps.rs`: Update/create step definitions for the new flow.

## Verification Plan

1.  Run `cargo test --test <new_bdd_test>` to ensure features are picked up.
2.  Observe failing tests due to missing UI elements or incorrect flow.
