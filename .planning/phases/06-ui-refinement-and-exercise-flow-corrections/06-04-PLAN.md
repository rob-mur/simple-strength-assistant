# Plan 06-04: Streamlined Workout Flow Implementation

## Goal
Streamline the workout flow by removing the redundant "Start Session" view in the Workout tab and starting sessions directly from exercise selections in the Library.

## Requirements
- Remove `StartSessionView` from `WorkoutView`.
- In `WorkoutView`, show an empty state when no session is active (e.g., "Go to Library to start a session").
- Add a "Start Session" button to each exercise's detail view in the Library.
- Clicking "Start Session" in the Library should switch the user to the Workout tab and begin the session.

## Success Criteria
1.  Workout tab defaults to a clear, user-friendly empty state when no session is active.
2.  Users can start a workout session from any exercise in the Library tab.
3.  Starting a session automatically switches the active tab to "Workout".
4.  The active session state is correctly initialized and preserved.

## Proposed Changes

### 1. `src/components/workout_view.rs`
- Update `WorkoutView` to display a message or button redirecting to the Library when `current_session` is `None`.

### 2. `src/components/library_view.rs`
- Add a "Start Session" button to each exercise in the list or exercise detail view.
- Trigger `WorkoutStateManager::start_session` and switch the `active_tab` to `Tab::Workout` upon clicking.

### 3. `src/app.rs`
- Ensure `active_tab` signal is accessible and can be changed from child components (e.g., via a context or event).

## Verification Plan
1.  Open the Library tab.
2.  Find an exercise and click its "Start Session" button.
3.  Verify the app automatically switches to the Workout tab and begins the session.
4.  Switch back to the Library and ensure the session is still active in the background.
5.  Complete the session and verify the Workout tab returns to the "No active session" empty state.
