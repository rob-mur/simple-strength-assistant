# Plan 06-02: Navigation Bar Polish and Safe Area Support

## Goal

Improve the bottom navigation bar's positioning and styling to ensure it handles mobile safe areas (notches/home indicators) correctly and provides a better user experience across different screen sizes.

## Requirements

- Use Tailwind CSS safe area utility classes or CSS `env(safe-area-inset-bottom)`.
- Ensure the bar is always fixed at the bottom.
- Add appropriate padding to prevent content overlap.
- Ensure the bar is visible on different mobile screen heights.

## Success Criteria

1.  Navigation bar has bottom padding for safe areas on mobile devices.
2.  The `main` content area has consistent padding to avoid overlap.
3.  The bar remains fixed at the bottom of the screen regardless of content height.

## Proposed Changes

### 1. `src/components/tab_bar.rs`

- Add `pb-[env(safe-area-inset-bottom)]` or equivalent DaisyUI/Tailwind padding to `TabBar`.
- Adjust overall height and layout of the tab buttons for better touch targets.

### 2. `src/app.rs`

- Update `main` container padding to dynamically handle the tab bar height and safe areas.
- Ensure `min-h-screen` and `flex-col` structure is consistent.

## Verification Plan

1.  Run the application in a mobile browser/emulator (e.g., iOS Safari or Android Chrome with a notch).
2.  Verify the tab bar is not obscured by the system home indicator or notch.
3.  Verify the "Workout" and "Library" tabs are easy to tap.
4.  Run Playwright tests (if applicable) to check for navigation bar visibility.
