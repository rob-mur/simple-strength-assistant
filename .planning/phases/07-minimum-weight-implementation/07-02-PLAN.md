# Plan 07-02: Suggestion Engine using Previous Session Weight

## Objective

Update the `Database` and `WorkoutState` to fetch the previous session's weight for an exercise when calculating initial predictions for a new session, falling back to the configured minimum weight if no previous sessions exist. This satisfies SUGG-01 and SUGG-02.

## Tasks

<tasks>
  <task id="7-02-01" autonomous="true">
    <description>Add database query for most recent set of an exercise.</description>
    <requirements>SUGG-01</requirements>
    <instructions>
      1. Open `src/state/db.rs`.
      2. Add a new async method `pub async fn get_last_set_for_exercise(&self, exercise_id: i64) -> Result<Option<crate::models::CompletedSet>, DatabaseError>`.
      3. Implement the query joining `sessions` and `sets` to get the most recent set by timestamp or ID for the given exercise ID. Ensure it returns a `CompletedSet` or parsing of a single set's data.
      4. Note: If `sessions` only has `exercise_name`, we may need to query by `exercise_name` or `exercise_id` depending on the current DB schema. Looking at `db.rs` line 258 `INSERT INTO exercises (name...)`, use whatever identifier is present in `sessions` table.
      5. Add corresponding test in `src/state/db_tests.rs`.
    </instructions>
  </task>

  <task id="7-02-02" autonomous="true">
    <description>Update `calculate_initial_predictions` to accept last set and use its weight.</description>
    <requirements>SUGG-01, SUGG-02</requirements>
    <instructions>
      1. In `src/state/workout_state.rs`, update `calculate_initial_predictions` to accept an `Option<&crate::models::CompletedSet>` as a second parameter.
      2. If a last set is provided and the exercise is weighted, use the last set's weight (potentially applying progression logic similar to `calculate_next_predictions`, or just the exact weight). The requirement says "most recent recorded weight". So just return that exact weight.
      3. If no last set is provided, fall back to `min_weight`.
      4. Update unit tests in `src/state/workout_state.rs`.
    </instructions>
  </task>

  <task id="7-02-03" autonomous="true">
    <description>Update `start_session` to fetch last set and calculate predictions.</description>
    <requirements>SUGG-01</requirements>
    <instructions>
      1. In `src/state/workout_state.rs`, modify `start_session`.
      2. Since it has `exercise.id` or `exercise.name`, call `db.get_last_set_for_exercise` (or `by_name` if needed).
      3. Pass the result into `calculate_initial_predictions`.
      4. Ensure all database errors during suggestion fetch are handled gracefully (fallback to no last set if DB query fails).
    </instructions>
  </task>
</tasks>

## Verification

- [ ] `cargo test` passes.
- [ ] A new session for an exercise with previous history correctly suggests the weight from the last set of the previous session.
- [ ] A new session for an exercise with no history suggests the configured minimum weight.
