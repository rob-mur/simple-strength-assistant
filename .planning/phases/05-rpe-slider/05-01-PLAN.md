# Plan: RPE Slider Implementation (Phase 5)

## Objective
Replace the current numeric RPE input with a custom, tactile discrete slider component.

## Requirements
- RPE-01: Adjust RPE via slider (1-10)
- RPE-02: Snap to 0.5 increments
- RPE-03: Prominent value display
- SUCCESS: Slider operates smoothly and snaps accurately.
- SUCCESS: Value display updates instantly.

## Proposed Changes

### 1. Component Creation
- Create `src/components/rpe_slider.rs`.
- Implement `RPESlider` component using Dioxus and pointer events.
- Features:
    - Horizontal track with 19 snap points (1.0 to 10.0 in 0.5 steps).
    - Visual indicators for whole numbers.
    - Color coding based on intensity (Green to Red).
    - Touch-optimized hitboxes.

### 2. Component Registration
- Update `src/components/mod.rs` to export `rpe_slider`.

### 3. UI Integration
- Modify `src/app.rs`:
    - Change `rpe_input` signal from `String` to `f64`.
    - Replace `input[type="number"]` with `RPESlider`.
    - Ensure `log_set` correctly handles the `f64` to `f32` conversion.

### 4. Style Refinement
- Ensure the slider matches the "Tape Measure" aesthetic.
- Add labels and ticks for better orientation.

## Verification Plan

### Automated Tests
- Create unit tests for `RPESlider` logic if possible (snapping math).
- Verify `CompletedSet` validation still passes with the new input method.

### Manual Verification
- Open the app in a browser.
- Start a session.
- Interact with the RPE Slider:
    - Drag the thumb.
    - Tap on different points.
    - Verify it snaps to 0.5 increments.
    - Verify the color changes as expected.
    - Log a set and verify the RPE is correctly saved in the "Completed Sets" table.
